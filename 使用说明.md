# 🎉 快速开始

## ✅ 安装依赖（首次运行）

```bash
npm install --legacy-peer-deps
```

## 🚀 启动系统

### Windows 用户

#### 方式一：双击启动（最简单）
```
双击 start.bat 文件
```

#### 方式二：命令行启动
```bash
node server.js        # 终端1：启动后端
npm run dev:fe      # 终端2：启动前端
```

### Ubuntu/Linux 用户

```bash
chmod +x start.sh      # 添加执行权限
./start.sh             # 一键启动
```

## 🌐 访问地址

- **前端页面**：http://localhost:5173
- **后端 API**：http://localhost:3001

## 📱 使用字体功能

### 方法1：复制文件 + 刷新（推荐）

1. 将字体文件复制到 `public/fonts/` 文件夹
2. 打开浏览器访问 http://localhost:5173
3. 进入：配置 → 全局设置 → 字体管理
4. 点击"刷新字体列表"按钮
5. 完成！字体会显示在"项目文件夹字体"区域

### 方法2：直接上传（更简单）

1. 打开字体管理页面
2. 点击"上传字体"按钮
3. 选择字体文件
4. 自动保存到服务器并更新列表
5. 完成！

## ⚙️ 端口说明

- 前端：5173
- 后端 API：3001

**端口冲突？** 修改 `server.js`：
```javascript
const PORT = process.env.PORT || 3001; // 改为其他端口
```

## 🔧 常见问题

### Q: npm install 失败
A: 使用以下命令：
```bash
npm install --legacy-peer-deps
```

### Q: 找不到字体
A: 检查以下几点：
1. 确保后端服务器正在运行（检查是否有 "Server started" 日志）
2. 确保字体文件在 `public/fonts/` 文件夹
3. 点击"刷新字体列表"按钮
4. 查看浏览器控制台（F12）是否有错误

### Q: 点击刷新没反应
A: 检查后端是否运行
```bash
# 检查端口
netstat -ano | findstr :3001
```

### Q: Ubuntu 部署
A: 按以下步骤：
```bash
npm install --legacy-peer-deps
node server.js &          # 后台启动后端
npm run dev:fe          # 启动前端
```

## 📦 部署到生产环境
