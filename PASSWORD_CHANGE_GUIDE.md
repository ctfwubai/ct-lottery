# 🔄 修改登录密码使用指南

## 📌 快速开始

### 方法一：通过系统设置页面修改（最简单）✨

**适用于：** 单机部署、个人使用、临时修改密码

**步骤：**

1. 登录系统（使用默认账号 `admin` / 密码 `admin123`）
2. 点击右上角【配置】按钮进入配置页面
3. 滚动到页面底部
4. 点击齿轮图标 ⚙️ 进入高级设置
5. 输入高级设置密码（默认：`admin888`）
6. 点击右上角紫色的 **🔑 修改登录密码** 按钮
7. 在弹出的对话框中填写：
   - **当前密码**：admin123（或你当前使用的密码）
   - **新密码**：至少6位字符
   - **确认新密码**：再次输入新密码
8. 点击【确认修改】
9. 修改成功后，下次登录使用新密码即可

---

### 方法二：通过环境变量配置（适合批量部署）

**适用于：** 生产环境、批量部署、需要预设密码

**步骤：**

1. 编辑 `.env.production` 文件
2. 修改以下配置：
   ```bash
   VITE_AUTH_USERNAME=your_username
   VITE_AUTH_PASSWORD=your_strong_password
   ```
3. 重新构建项目：
   ```bash
   pnpm build
   ```
4. 部署到服务器

---

## ⚠️ 重要提示

### 两种方法的区别

| 特性 | 方法一（系统设置） | 方法二（环境变量） |
|------|-------------------|-------------------|
| 优点 | 简单快捷，无需重新构建 | 硬编码密码，适合批量部署 |
| 缺点 | 只在当前浏览器生效 | 需要重新构建和部署 |
| 适用场景 | 个人使用、临时修改 | 生产环境、多用户场景 |
| 清除缓存后 | 恢复默认密码 | 保持新密码 |

### 推荐使用方式

1. **个人/单机部署**：使用方法一，简单方便
2. **生产环境/多人使用**：使用方法二，统一密码管理
3. **临时修改密码**：使用方法一
4. **永久修改密码**：使用方法二

---

## 🔐 密码安全建议

1. **使用强密码**：至少8位，包含大小写字母、数字和特殊字符
2. **定期更换**：建议每3个月更换一次
3. **不要使用默认密码**：首次使用后立即修改
4. **不要重复使用**：不要与其他系统使用相同密码
5. **保管好密码**：记录在安全的地方

## ❓ 常见问题

### Q: 修改密码后其他电脑无法登录？

A: 方法一修改的密码只保存在当前浏览器。如果需要其他电脑也能登录，请：
- 在其他电脑上分别修改密码
- 或使用方法二（环境变量）统一配置

### Q: 忘记密码了怎么办？

A: 清除浏览器 localStorage 可以恢复默认密码：

1. 按 F12 打开开发者工具
2. 切换到 【Application】标签
3. 左侧找到【Local Storage】
4. 右键点击你的网站地址
5. 选择【Clear】
6. 刷新页面，使用默认密码登录

### Q: 清除浏览器缓存后密码会变吗？

A: 是的，方法一修改的密码会恢复到默认值。如果需要持久化，请使用方法二（环境变量配置）。

### Q: 可以设置多个账号吗？

A: 当前版本只支持单一账号密码认证。
